US配列ユーザー向け / ⌘英かな代替

左右⌘の単押しで
「かな / 英数」を切り替える

かなCmdは、US配列キーボード環境で左右の Command（⌘）キー単押しにより入力ソース（かな / 英数）を切り替える macOSメニューバー常駐アプリです。macOSアップデート後に既存ツールが不安定になったときの代替にも。

⬇︎ Download（最新版） GitHubを見る

※ 初回起動では「右クリック → 開く」が必要な場合があります（Gatekeeper）。

ワンポイント

対象 US配列 / 英語⇄日本語の切替に困る人
形式 メニューバー常駐（軽量）
切替 左右⌘単押し（誤爆しにくい）
補助 割当変更・ログイン時起動・診断
左⌘ ⌘ → かな
右⌘ ⌘ → 英数

アプリ内「診断」で状態確認できます。

できること

左右⌘の単押しで切替

Commandキー単押しだけを検知し、かな/英数へ切り替えます。ショートカットの邪魔をしにくい設計。

割り当ての入れ替え

左⌘=かな／右⌘=英数 をワンタップで反転。環境に合わせて調整できます。

ログイン時起動

ログイン時に自動起動できるので、常用でも忘れません（macOS 13+）。

Dock表示の切替

Dockに表示/非表示を選べます。常駐ツールっぽくスッキリ運用できます。

診断メニュー

権限状態（入力監視/アクセシビリティ）やEventTapの状態を確認でき、トラブル時の切り分けが楽。

軽量・シンプル

余計な設定を増やさず、必要な機能だけに絞ったミニマル設計。

インストール

手順

  1. Releasesから 最新版zip をダウンロード
  2. zipを展開し、かなCmd.app を Applications（アプリケーション） へ移動
  3. 初回起動時にブロックされたら：Finderでアプリを右クリック → 開く
  4. 権限を許可：入力監視 と アクセシビリティ
macOSの「システム設定 → プライバシーとセキュリティ」から設定できます。 許可後、アプリ再起動で反映されることがあります。

権限（必須）

入力監視（Input Monitoring） システム設定 → プライバシーとセキュリティ → 入力監視 → かなCmd をON アクセシビリティ（Accessibility） システム設定 → プライバシーとセキュリティ → アクセシビリティ → かなCmd をON ※ アプリ内メニュー「診断：状態を見る」で確認できます。
Tip: Xcodeから起動したビルドと /Applications のアプリを混ぜると、権限がズレる場合があります。 ふだん使いは /Applications から起動が安定です。

FAQ

キーを押しても切り替わりません

まず「入力監視」と「アクセシビリティ」がONか確認してください。アプリ内「診断：状態を見る」も有効です。 それでもダメなら、アプリを終了→再起動、macOS再起動を試してください。

権限をONにしたのに「未許可」と表示されます

Xcodeのデバッグ実行と /Applications の実体が混在している可能性があります。 いま動いているアプリの実体（パス）を揃え、権限一覧で一度OFF→ONし直してください。

セキュリティ警告で起動できません

Finderでアプリを右クリック→「開く」。または「システム設定 → プライバシーとセキュリティ」下部の許可ボタンから開いてください。 （署名/公証対応は将来的に検討できます）

⌘ショートカットと干渉しませんか？

かなCmdは「⌘単押し」を主に検知し、⌘＋他キーが押された場合は切替しないように作るのが一般的です。 万一気になる挙動があればIssueで報告してください（OS/キーボード環境も併記）。
