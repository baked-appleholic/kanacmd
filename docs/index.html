US配列ユーザー向け / ⌘英かな代替

左右⌘の単押しで
「かな / 英数」を切り替える

かなCmdは、US配列キーボード環境で左右の Command（⌘）キー単押しにより入力ソース（かな / 英数）を切り替える macOSメニューバー常駐アプリです。

⬇︎ Download（最新版） GitHubを見る

※初回起動でブロックされた場合も、手順通りに許可すれば起動できます（下に詳しく記載）。

こんな人におすすめ

US配列で困ってる かな/英数切替が面倒・安定しない
⌘英かな代替が欲しい アップデート後に固まる/反応しない
ショートカットを守りたい ⌘単押しを狙って切替（誤爆抑制）
常駐は軽くしたい メニューバーだけで完結
対応OS：macOS 13+ ログイン時起動：macOS 13+

特徴

左右⌘の単押しで切替

Commandキー単押しだけを検知して、かな/英数へ切り替えます。

割り当ての入れ替え

左⌘=かな／右⌘=英数 をワンタップで反転できます。

ログイン時起動（macOS 13+）

ログイン時に自動起動。メニューバーに常駐してすぐ使えます。

Dock表示の切り替え

Dockに表示/非表示を選べます。常駐ツールとしてスッキリ運用。

診断メニュー

権限（入力監視/アクセシビリティ）やEventTap状態を表示して切り分けが楽。

軽量・シンプル

必要機能だけに絞ったミニマル設計。迷わず使えます。

インストール

1) 最新版（Releases/latest） から zip をダウンロード
2) 展開して かなCmd.app を Applications（アプリケーション） に移動
3) 起動後、入力監視 と アクセシビリティ の権限を許可
4) うまく動かない場合はアプリ再起動／macOS再起動

必要な権限（重要）

  • 入力監視（Input Monitoring）
    システム設定 → プライバシーとセキュリティ → 入力監視 → かなCmd をON
  • アクセシビリティ（Accessibility）
    システム設定 → プライバシーとセキュリティ → アクセシビリティ → かなCmd をON
アプリ内メニュー 「診断：状態を見る」 で確認できます。

初回起動でブロックされる場合（Gatekeeper）

本アプリは現時点で「署名/公証（Notarization）」が未対応のため、初回起動時に macOS によりブロックされる場合があります。
これはマルウェア検出ではなく、Appleが開発元を検証できない（未公証）ことによる警告です。

  • システム設定 → プライバシーとセキュリティ → 画面下部の 「このまま開く」
  • または、Finderでアプリを右クリック（control+クリック）→ 開く

FAQ

キーを押しても切り替わりません

まず「入力監視」と「アクセシビリティ」がONか確認してください。アプリ内「診断：状態を見る」でも確認できます。アプリ再起動やmacOS再起動も有効です。

権限をONにしたのに「未許可」と表示されます

Xcode実行と /Applications の実体が混在している可能性があります。起動しているアプリの実体を揃え、権限一覧で一度OFF→ONし直してください。

セキュリティ警告で起動できません

システム設定 → プライバシーとセキュリティ の下部にある「このまま開く」から許可できます。Finderで右クリック→開くでも起動できます。

配布がもっと楽になる方法は？

Developer ID署名 + Notarization（公証）を行うと、警告が出にくくなり配布が楽になります（Apple Developer Programが必要）。
